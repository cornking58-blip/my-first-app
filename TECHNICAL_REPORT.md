## 1. Общее описание продукта

### 1.1 Назначение
**bAIkov** — мобильное приложение-справочник зарегистрированных гербицидов Российской Федерации. Предназначено для агрономов, фермеров и специалистов по защите растений. Позволяет быстро находить препараты, просматривать полные карточки с регламентами применения и проводить продвинутое сравнение двух гербицидов по действующим веществам и стоимости.

### 1.2 Ключевые функции MVP
| Функция | Статус | Описание |
|---------|--------|----------|
| Поиск гербицидов | ✅ Готово | Полнотекстовый поиск по названию, культуре, ДВ, производителю |
| Карточка препарата | ✅ Готово | Полная информация + все регламенты применения (аккордеон) |
| Сравнение двух препаратов | ✅ Готово | Анализ ДВ, дозировок, механизмов действия |
| Калькулятор стоимости | ✅ Готово | Расчёт стоимости на 1 га и 1 г ДВ |
| Фильтр «Только действующие» | ✅ Готово | Фильтрация по статусу регистрации |
| Статистика базы | ✅ Готово | Счётчики препаратов, регистраций, применений |

### 1.3 Целевая аудитория
- Агрономы-практики (основная ЦА)
- Дистрибьюторы СЗР
- Консультанты по защите растений
- Студенты агрономических вузов

### 2.5 Группировка продуктов
Продукты группируются по `product_key = "{product_name}|{registration_number}"`. Один продукт может иметь множество записей (применений) — на разные культуры и целевые объекты. Именно поэтому 956 уникальных продуктов дают 3 232 записи.

## 4. API — Эндпоинты бэкенда

### 4.1 Список эндпоинтов

| # | Метод | Путь | Назначение |
|---|-------|------|------------|
| 1 | GET | `/api/health` | Проверка здоровья сервиса + статистика |
| 2 | POST | `/api/admin/import-excel` | Импорт данных из Excel |
| 3 | GET | `/api/herbicides/search` | Поиск гербицидов |
| 4 | GET | `/api/herbicides/{product_key}` | Карточка продукта |
| 5 | POST | `/api/herbicides/compare` | Базовое сравнение |
| 6 | POST | `/api/herbicides/compare-advanced` | Продвинутое сравнение + калькулятор |
| 7 | GET | `/api/stats` | Статистика базы |

### 4.2 Детальное описание

#### `GET /api/health`
**Ответ:**
```json
{
  "status": "healthy",
  "database": "connected",
  "records_count": 3232,
  "timestamp": "2026-04-01T11:15:42.963778"
}
```

#### `GET /api/herbicides/search`
**Параметры:**
- `q` (string, опц.) — поисковый запрос (regex, case-insensitive)
- `only_active` (bool, опц.) — только действующие регистрации
- `limit` (int, макс 200, по умолчанию 50) — лимит результатов

**Поиск по полям:** product_name, crop, target_object, active_substances_raw, manufacturer

**Агрегация:** Группировка по `product_key` с подсчётом `applications_count`

**Время ответа:** 8–16 мс

#### `GET /api/herbicides/{product_key}`
**Ответ:** Полная карточка продукта со всеми применениями (массив `applications`).

#### `POST /api/herbicides/compare-advanced`
**Тело запроса:**
```json
{
  "left_key": "Препарат A|001-01-001-1",
  "right_key": "Препарат B|002-02-002-1",
  "left_price": 1500.0,    // опционально, ₽/л или ₽/кг
  "right_price": 2000.0    // опционально
}
```

**Алгоритм сравнения:**
1. Парсинг действующих веществ из сырой строки (regex)
2. Разделение на активные ДВ и антидоты
3. Нахождение максимальной нормы расхода (л/га)
4. Сопоставление одинаковых ДВ (по нормализованным именам)
5. Сопоставление сходных по механизму действия (9 категорий)
6. Выделение уникальных ДВ каждого препарата
7. Расчёт г/га для каждого ДВ при макс. норме
8. Расчёт стоимости: ₽/га, ₽/г ДВ, вклад каждого ДВ в стоимость

**Время ответа:** ~5 мс

## 5. Алгоритмы и бизнес-логика

### 5.1 Парсинг действующих веществ (`parse_active_substances`)
Входная строка типа: `(360 г/л Глифосат кислоты (изопропиламинная соль))`

**Алгоритм:**
1. Удаление внешних скобок
2. Разделение по `+` на компоненты
3. Для каждого компонента: regex `(\d+[.,]?\d*)\s*(г/л|г/кг)\s*(.+)`
4. Определение антидотов по ключевому слову «антидот»
5. Очистка имени от префикса «антидот –»
6. Возврат: `{name, concentration, unit, is_antidote}`

### 5.2 Категоризация ДВ (`get_substance_category`)
9 категорий по механизму действия:

| # | Категория | Примеры ДВ |
|---|-----------|------------|
| 1 | Ауксиноподобные (гормональные) | 2,4-Д, дикамба, клопиралид |
| 2 | Ингибиторы ALS (сульфонилмочевины) | трибенурон, метсульфурон, римсульфурон |
| 3 | Ингибиторы ACCase (граминициды) | феноксапроп, хизалофоп, клетодим |
| 4 | Глифосаты (EPSP-ингибиторы) | глифосат |
| 5 | Ингибиторы PPO | карфентразон, оксифлуорфен |
| 6 | Ингибиторы фотосистемы II | метрибузин, атразин, прометрин |
| 7 | Хлорацетамиды (почвенные) | метолахлор, ацетохлор |
| 8 | Ингибиторы HPPD | мезотрион, темботрион |
| 9 | Антидот (сэйфнер) | клоквинтосет, мефенпир |

### 5.3 Парсинг нормы расхода (`parse_rate_max`)
Извлекает максимальное числовое значение из строки нормы расхода:
- `"0,6-0,8"` → `0.8`
- `"1,5"` → `1.5`

### 5.4 Калькулятор стоимости
**Формулы:**
- **Стоимость 1 га** = Цена за л/кг × Макс. норма расхода (л/га)
- **Стоимость 1 г ДВ** = Цена за л/кг ÷ Суммарная концентрация ДВ (г/л)
- **Вклад ДВ (%)** = Концентрация ДВ ÷ Суммарная концентрация × 100

## 10. Известные ограничения и технический долг

### 10.1 Архитектурные
| # | Проблема | Влияние | Приоритет |
|---|----------|---------|-----------|
| 1 | Монолитный `server.py` (841 строка) | Усложняет поддержку | P1 |
| 2 | Нет индексов MongoDB | Замедлит при росте данных | P1 |
| 3 | Нет кеширования | Лишние запросы к БД | P2 |
| 4 | Regex-поиск вместо полнотекстового | Менее эффективен на больших объёмах | P2 |

### 10.2 Функциональные
| # | Ограничение | Описание |
|---|-------------|----------|
| 1 | Нормализация данных | Разные написания одних и тех же культур/сорняков |
| 2 | Статусы с датами | Статусы вида «Не действует, 18.03.2025» не парсятся |
| 3 | Нет оффлайн-режима | Данные не кешируются локально |
| 4 | Нет истории сравнений | Результаты сравнений не сохраняются |

## 12. Инструкции по развёртыванию и эксплуатации

### 12.1 Переменные окружения

**Backend (`/app/backend/.env`):**
```
MONGO_URL=mongodb://localhost:27017
DB_NAME=herbicides_db
```

**Frontend (`/app/frontend/.env`):**
```
EXPO_PUBLIC_BACKEND_URL=https://<domain>.preview.emergentagent.com
```

### 12.2 Запуск сервисов
```bash
# Backend
sudo supervisorctl restart backend

# Frontend (Expo)
sudo supervisorctl restart expo
```

### 12.3 Импорт данных
```bash
curl -X POST http://localhost:8001/api/admin/import-excel \
  -F "file=@herbicides_raw_FINAL_checked.xlsx"
```
**Внимание:** Импорт полностью заменяет все существующие данные (delete_many + insert_many).

### 12.4 Проверка здоровья
```bash
curl http://localhost:8001/api/health
# Ожидаемый ответ: {"status":"healthy","database":"connected","records_count":3232,...}
```

### 12.5 Маршрутизация (Kubernetes)
- Все запросы на `/` → порт 3000 (Expo frontend)
- Все запросы на `/api/*` → порт 8001 (FastAPI backend)
